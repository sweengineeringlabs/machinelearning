# Key Quantization Concepts

This document explains the foundational concepts used in the `quant-rs` tool.

---

## 1. NormalFloat 4 (NF4)
Standard 4-bit integers (`INT4`) use 16 evenly spaced points (e.g., -8, -7, ... 0, ... 7). However, neural network weights aren't "even"—they usually follow a **Normal Distribution** (a bell curve), where most values are clustered near zero.

- **The NF4 Innovation:** Instead of wasting points on values that rarely occur, NF4 uses a specialized lookup table where the 16 available points are **densely packed near zero** and spread out at the edges.
- **The Result:** It captures significantly more "information" about the model's weights than standard 4-bit integers can, maintaining higher model "intelligence" at a significantly reduced size.

---

## 2. Block-wise Quantization
In basic quantization, a single "Maximum" value is found for an entire weight tensor (which may contain millions of parameters) and used as a global scale factor.

- **The Outlier Problem:** If a single weight is an outlier (e.g., 100.0) while the rest are tiny (e.g., 0.01), the global scale becomes so large that all the small, important weights are rounded to zero during quantization.
- **The Block-wise Solution:** The weight tensor is divided into small, independent "blocks" (typically 64 or 128 elements). Each block is assigned its own unique scale factor.
- **The Result:** Outliers only affect the precision of their specific block, allowing the rest of the model to maintain high-fidelity scaling. This is a critical technique for Large Language Models (LLMs).

---

## 3. Combined Impact
When **NF4** is combined with **Block-wise** scaling:
1. **Extreme Compression:** 32-bit weights are reduced to 4 bits (**8x reduction** in memory).
2. **High Precision:** The specialized NF4 mapping preserves the distribution of the weights.
3. **Robustness:** Block-wise scaling prevents outlier weights from degrading the entire model's performance.
