## 1. Interface Analysis: WeightScope (v2.5.0)
The `weights_analyzer.html` is a high-fidelity, single-page application (SPA) designed to visualize and simulate the impact of quantization on neural network architectures.

### **Core Capabilities**
*   **Multi-Format Support:** Pre-configured for PyTorch, GGUF, and SafeTensors.
*   **Live Metrics:** Real-time calculation of parameters, model size (MB/GB), and compression ratios.
*   **Granular Visualization:** 
    *   **Layer Heatmap:** A magnitude-based grid for identifying "hot" and "cold" weights.
    *   **Distribution Charts:** Histogram and Scatter plots for statistical weight analysis.
    *   **Sensitivity Mapping:** Identifies layers most prone to accuracy loss (e.g., `Embedding`).

### **Design Architecture**
*   **UI/UX:** Modern dark-mode dashboard using CSS Grid and Flexbox.
*   **Engine:** Pure JavaScript simulation for consistent data generation across different "Sample Models."
*   **Extensibility:** Modular functions for exporting results to JSON, CSV, or mock checkpoints.

## 2. Quantization Fundamentals
Quantization is the process of mapping high-precision continuous values (32-bit floats) to a lower-precision discrete set (8-bit or 4-bit integers).

### **Precision Levels**
1.  **FP32 (Full):** 4 bytes/param. Highest accuracy, high memory footprint.
2.  **FP16/BF16 (Half):** 2 bytes/param. Industry standard for modern training.
3.  **INT8 (8-bit):** 1 byte/param. Standard for mobile/edge production.
4.  **INT4/NF4 (4-bit):** 0.5 bytes/param. Enables LLMs on consumer hardware.

### **Key Metrics & Trade-offs**
*   **Compression Ratio:** Factor of model size reduction (e.g., FP32 to INT8 = 4.0×).
*   **Perplexity (PPL) Delta:** Measure of quality loss; lower is better.
*   **Sparsity:** Percentage of zero or near-zero weights, used for acceleration.

## 3. Tooling & Ecosystem
The following tools represent the current state-of-the-art in quantization:

| Category | Primary Tools | Use Case |
| :--- | :--- | :--- |
| **Development** | PyTorch, TensorFlow, Brevitas | Training-time quantization (QAT) and R&D. |
| **LLM Compression** | AutoGPTQ, AutoAWQ, Bitsandbytes | Shrinking massive models for local or cloud usage. |
| **Local Inference** | llama.cpp (GGUF), ExLlamaV2 (EXL2) | Running models on Mac/PC with minimal hardware. |
| **Cloud/Production** | vLLM, NVIDIA TensorRT-LLM | High-throughput, multi-user deployments. |
| **Mobile/Edge** | TFLite, CoreML, OpenVINO | Optimizing for Android, iOS, and Intel hardware. |
